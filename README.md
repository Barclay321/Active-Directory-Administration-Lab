# Active Directory Administration Lab

## Objective
This lab demonstrates hands-on experience with Active Directory administration in a Windows domain environment. It focuses on user management, computer management, group policies, and authentication methods.

---

## Skills Learned
- Active Directory structure and domain concepts
- User account management
- Computer and domain-joined device management
- Organizational Units (OUs)
- Group Policy Objects (GPOs)
- Authentication methods (Kerberos and NTLM)
- Understanding domains, trees, forests, and trusts

---

## Tools Used
- Windows Active Directory
- Active Directory Users and Computers (ADUC)
- Group Policy Management Console (GPMC)

---

## Lab Activities

### Active Directory Overview
Learned the purpose of Active Directory and how it centralizes identity and resource management in a Windows domain.

### User Management
Created and managed user accounts within Active Directory.

### Computer Management
Reviewed and managed domain-joined computers within organizational units.

### Group Policies
Explored Group Policy Objects (GPOs) and how they are used to enforce security settings across a domain.

### Authentication Methods
Studied Kerberos and NTLM authentication and how they function within a domain environment.

## Key Takeaways
- Active Directory is the core identity management system in enterprise networks.
- Group Policies allow centralized control of security and system configurations.
- Kerberos is the primary authentication protocol in modern Windows domains.
- Proper organization of users and computers improves security and scalability.
